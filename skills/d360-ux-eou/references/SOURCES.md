# Reference Sources

This directory contains adapted versions of Data 360 UX documentation maintained in Google Slides. These files are used by the d360-ux-eou skill to evaluate UI/UX against Data 360 design standards.

## Documents

### Simplicity Scale
- **Source**: [Data 360 Simplicity Scale (Google Slides)](https://docs.google.com/presentation/d/1oWuMrwlamlTBedNsUbBj-q6Xn4Ie1Y4_2Zou8VdE7eQ/edit?slide=id.g38e250adfd8_16_539#slide=id.g38e250adfd8_16_539)
- **Local file**: `simplicity-scale.md`
- **Last synced**: 2026-03-31
- **Purpose**: Assessment criteria for evaluating features against the L1/L2/L3 simplicity framework

### Experience Principles
- **Source**: [Data 360 Experience Principles (Google Slides)](https://docs.google.com/presentation/d/1cQCPANXQ7jfX5AJ9kOXRingF4mRvFsl2L9EvLGjDFzE/edit?slide=id.g2f42db121f4_2_482#slide=id.g2f42db121f4_2_482)
- **Local file**: `experience-principles.md`
- **Last synced**: 2026-03-31
- **Purpose**: Evaluation rubric for the 6 Data 360 Experience Principles (P1-P6)

## Keeping Files in Sync

When the source Google Slides are updated:

1. Review the changes in the source deck
2. Update the corresponding local markdown file
3. Update the "Last synced" date in both the file header and this SOURCES.md
4. Test the d360-ux-eou skill to ensure the changes work as expected
